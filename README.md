**Git Convention**

## Git  Commit Convention
- 제목과 본문을 빈 행으로 구분합니다.
- 제목을 50글자 이내로 제한합니다.
- 제목의 첫 글자는 대문자로 작성합니다.
- 제목의 끝에는 마침표를 넣지 않습니다.
- 제목은 명령문으로! 과거형을 사용하지 않습니다.
- 본문의 각 행은 72글자 내로 제한합니다.
- 본문은 어떻게 보다는 무엇과 왜를 설명합니다.

## 커밋메세지 규칙
```
{태그} : 작업내용(#{이슈번호})
<BLANK LINE> 
- 세부 작업 내용1
- 세부 작업 내용2
<BLANK LINE> 
(해결,참고,관련): #{이슈번호}
```
## 이슈 작성
- 이슈제목: {구현 내용} API - {API url}

## Branch 작성방법
- {태그}#{이슈번호}-seonghyeon

## git commit
- 이슈브랜치 ➡️ dev ➡️ main
